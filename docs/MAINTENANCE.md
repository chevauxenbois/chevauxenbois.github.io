# Maintaining Chevauxenbois

This site is a plain static website. It does not depend on Codex, a database,
paid hosting, or a CMS. You can update it with any editor and deploy it with
GitHub Pages.

## Folder Map

```text
content/writing/     Blog posts as Markdown
content/projects/    Project pages as Markdown
content/french/      Language notes as Markdown
public/images/       Images used by the site
scripts/build.mjs    Static site generator
scripts/import-medium.mjs  Medium export importer
src/styles.css       Visual design
dist/                Generated website output
```

Do not edit files inside `dist/` directly. They are generated by `npm run build`.

## Add A New Blog Post

Create a new Markdown file under `content/writing/`.

Example:

```md
---
title: "Article title"
date: "2026-06-17"
summary: "One sentence summary for cards and SEO."
tags: ["Information Security", "GRC", "AI Governance"]
cover: "/images/articles/example.png"
canonical: "https://medium.com/@harshkahate/original-link-if-any"
---

Write the article here.

![Screenshot caption](/images/articles/example.png)
```

Then run:

```bash
npm run build
```

## Add Images

Put images in:

```text
public/images/articles/
public/images/projects/
public/images/brand/
```

Reference them from Markdown like this:

```md
![Caption](/images/articles/my-image.png)
```

For long-term ownership, prefer local image files over external CDN URLs.

## Import Medium Again

If you export Medium again, run:

```bash
npm run import:medium -- /path/to/medium-export.zip
npm run build
```

The importer removes previously imported Medium posts and recreates them from
the archive. It skips drafts and tiny reply posts.

## Publish With GitHub Pages

Typical flow after the repo is connected to GitHub:

```bash
git status
git add .
git commit -m "Update portfolio"
git push
```

GitHub Actions will build and publish the site if Pages is enabled for the
repository.

## Custom Domain

For `chevauxenbois.com`, GitHub Pages needs three things:

1. This repository must publish through GitHub Pages.
2. The built site must contain a `CNAME` file.
3. GoDaddy DNS must point the domain to GitHub Pages.

This site already includes:

```text
public/CNAME
```

with:

```text
chevauxenbois.com
```

The build copies that file into `dist/CNAME`.

### GitHub Pages Settings

In GitHub, open the repository:

```text
chevauxenbois/chevauxenbois.github.io
```

Then:

```text
Settings -> Pages
```

Use:

```text
Source: GitHub Actions
Custom domain: chevauxenbois.com
Enforce HTTPS: enabled after GitHub finishes certificate provisioning
```

### GoDaddy DNS

In GoDaddy:

```text
My Products -> chevauxenbois.com -> DNS -> Manage DNS
```

Set root domain `A` records:

| Type | Name | Value |
| --- | --- | --- |
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

Set `www`:

| Type | Name | Value |
| --- | --- | --- |
| CNAME | www | chevauxenbois.github.io |

Remove any old GoDaddy parking/default `A` records for `@` if they conflict.

### Verify DNS

After GoDaddy saves the records, DNS can take minutes to hours. Check with:

```bash
dig chevauxenbois.com +noall +answer -t A
dig www.chevauxenbois.com +noall +answer
```

The root domain should return GitHub's four `185.199...` IPs. `www` should
point to `chevauxenbois.github.io`.

## If You Stop Using Codex

Nothing breaks. The site is maintainable with:

- GitHub web editor for small text fixes.
- VS Code for normal editing.
- Any AI tool that can edit files.
- Manual Markdown editing.

The only required commands are:

```bash
npm run build
npm run serve
```

For deployment:

```bash
git add .
git commit -m "Your change"
git push
```
